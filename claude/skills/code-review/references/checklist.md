# レビューチェックリスト

変更箇所のみに注目すること。pre-existing issues は報告しない。

## 1. Security

- [ ] ユーザー入力がサニタイズ・バリデーションされているか（SQL, XSS, コマンドインジェクション）
- [ ] 認証・認可チェックが適切に行われているか
- [ ] シークレット（APIキー、トークン、パスワード）がハードコードされていないか
- [ ] 機密データがログ・エラーメッセージに含まれていないか
- [ ] CORS, CSRF, セキュリティヘッダーが適切か

## 2. Logic & Correctness

- [ ] null / undefined / 空配列 / 空文字列の取り扱いが正しいか
- [ ] 境界値（0, 負数, 最大値, 空コレクション）でのエッジケース
- [ ] レース条件・並行処理の安全性
- [ ] 状態遷移の一貫性（フラグの更新漏れ、不完全なロールバック）
- [ ] 条件分岐の網羅性（else 漏れ、switch の default）

## 3. Error Handling

- [ ] 空の catch ブロック（サイレントキャッチ）がないか
- [ ] エラーが適切に伝播・報告されているか
- [ ] 外部サービス呼び出しの失敗時の挙動が定義されているか
- [ ] リソースのクリーンアップ（ファイル、コネクション、ロック）

## 4. Performance

- [ ] N+1 クエリ、ループ内の不要な I/O
- [ ] 不要な再計算・再レンダリング
- [ ] メモリリーク（イベントリスナー、タイマー、サブスクリプションの解除漏れ）
- [ ] 大量データ処理時のページネーション・ストリーミング

## 5. Code Quality

- [ ] 命名が意図を明確に表現しているか
- [ ] 関数が単一責務を守っているか
- [ ] 重複コードが新たに導入されていないか
- [ ] テストが変更をカバーしているか（新機能にテストがあるか）
