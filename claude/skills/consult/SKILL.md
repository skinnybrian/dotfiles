# コンサルト

引数: $ARGUMENTS

## ペルソナ選択

ペルソナ一覧と思考特性は [references/personas.md](references/personas.md) を参照。

- 引数が空 → 利用可能なペルソナ一覧を表示して選択を促す
- 引数あり → 第1引数をペルソナ名とマッチ（部分一致可、複数候補時は確認）、残りをトピックとする
- マッチなし → 引数全体をトピックとし、内容から最適なペルソナを自動選択（理由を冒頭に明示）

## 処理フロー

1. プロジェクト内なら技術スタックを確認。外部情報が必要ならWebSearchで最新動向を1-2件取得
2. 選択したペルソナの専門性と思考様式になりきって分析（思考特性は [references/personas.md](references/personas.md) 参照）
3. 以下のフォーマットで報告:

```
> ペルソナ: [ペルソナ名]（自動選択の場合は理由も記載）

## [ペルソナ名]としての分析: [トピック]

### 現状評価
### 推奨事項
### リスクと注意点
### 次のステップ
- [ ] すぐにできるアクション
- [ ] 中期的に取り組むべき項目
- [ ] 長期的に検討すべき項目
### 参考情報
```
