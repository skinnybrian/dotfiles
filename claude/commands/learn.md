# Learn コマンド

引数: $ARGUMENTS

## 手順

1. **記録内容の決定**
   - 引数が指定されている場合はその内容を記録対象とする
   - 引数が省略されている場合は、セッション中の会話を振り返り、以下の観点で学びを抽出する：
     - 発見したパターンや規約
     - 解決した問題とその原因・対処法
     - ユーザーの好みやワークフロー
     - プロジェクト固有の重要な知識（アーキテクチャ、ファイル構成など）

2. **プロジェクト CLAUDE.md の確認**
   - 作業ディレクトリの `CLAUDE.md` を読み、既存の内容と重複がないか確認する
   - 既存のセクション構造を把握する

3. **追記内容の提示**
   - 追記する内容をユーザーに提示し、確認を取る
   - 既存のセクションに追記できる場合はそのセクションに、新しいトピックなら適切なセクション見出しとともに追記する

4. **プロジェクト CLAUDE.md に追記**
   - 確認後、Edit ツールで CLAUDE.md に追記する
   - 簡潔に、箇条書きで記載する（冗長な説明は避ける）

5. **グローバル追記の提案**
   - 記録した内容が特定プロジェクトに限らず汎用的に適用できるものであれば、グローバル CLAUDE.md（`~/.claude/CLAUDE.md`）への追記も提案する
   - ユーザーが同意した場合のみグローバルにも追記する

## 注意事項

- 既存の記述と矛盾する内容がある場合は、既存の記述を更新する（重複させない）
- 一時的な作業状態や未検証の情報は記録しない
- 簡潔さを重視する。1項目1-2行以内に収める
