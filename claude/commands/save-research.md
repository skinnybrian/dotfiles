# 調査結果保存コマンド

引数: $ARGUMENTS

## 手順

### 1. 保存対象の特定

現在の会話を振り返り、保存すべき調査結果を特定する:

- `/research-best-practices`、`/research`、`/consult` の実行結果
- プランモードでの調査・分析結果
- Agent による Web 検索・コード調査の結果
- その他、ユーザーが保存を指示した内容

引数が指定されている場合はそれをトピック名とする。省略時は調査内容からトピック名を自動生成する（短く、ファイル名に適した形式）。

### 2. 保存先の決定

以下の優先順で保存先を決定する:

1. 現在の作業ディレクトリに `docs/research/` があればそこに保存
2. なければ `docs/research/` を `mkdir -p` で作成して保存
3. 作業ディレクトリが `~/.claude/` の場合は `~/.claude/docs/research/` に保存

ファイル名: `YYYY-MM-DD_<topic-slug>.md`（日本語トピックはローマ字またはそのまま使用、スペースはハイフンに変換）

### 3. コンテンツの構造化

以下のテンプレートに従って調査結果を構造化する。会話中の情報を漏れなく含めること:

```markdown
---
title: "<トピック名>"
date: <YYYY-MM-DD>
tags:
  - <関連タグ1>
  - <関連タグ2>
command: "<実行したコマンド（/research-best-practices 等）>"
sources:
  - <参考URL1>
  - <参考URL2>
---

# <トピック名>

## サマリー

<調査結果の要約を2-3文で>

## 詳細

<調査結果の本文。セクション構造は元の出力に合わせる>

### アプローチ比較

<比較表がある場合はここに含める>

### 推奨事項

<推奨ベストプラクティスや具体的アクション>

## 学術的知見

<関連する論文や研究がある場合。なければこのセクションを省略>

## 参考リンク

- [タイトル](URL) — 概要
```

### 4. ファイルの書き出し

Write ツールでファイルを作成する。作成後、ファイルパスをユーザーに報告する。

### 5. 追加保存先の提案

ファイル作成後、以下を確認・提案する:

- **Obsidian**: MCP サーバー（mcp__obsidian）が利用可能な場合、「Obsidian Vault にも保存する？」と提案
- **Notion**: MCP サーバー（mcp__notion）が利用可能な場合、「Notion にも保存する？」と提案
- どちらも利用不可の場合は提案しない

## 注意事項

- 会話中のソース URL は可能な限りすべて収集する
- フロントマターの tags は調査内容に関連するキーワードを 3-5 個付ける
- Obsidian 互換を意識し、wikilinks ではなく標準 Markdown リンクを使う
- 画像や大きなコードブロックがある場合もそのまま含める
- 既に同名ファイルが存在する場合はユーザーに確認（上書き or 連番付与）
